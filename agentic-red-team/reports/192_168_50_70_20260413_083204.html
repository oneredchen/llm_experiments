Executive Summary

This penetration test was conducted on target 192.168.50.70, identified as Metasploitable2—a deliberately vulnerable Ubuntu 8.04 Linux system used for security training. The assessment covered reconnaissance, scanning, vulnerability analysis, exploitation, and post-exploitation phases.

The engagement revealed a critically insecure system with multiple severe vulnerabilities. Most notably, a pre-configured backdoor on port 1524 provided immediate root access to the system. Additional critical vulnerabilities included a vsftpd 2.3.4 backdoor (CVE-2011-2523) and numerous high-severity issues such as Samba null session authentication, UnrealIRCd backdoor (CVE-2010-2541), and vulnerable web applications (DVWA, phpMyAdmin, Mutillidae).

The system is fully compromised with root privileges. The attacker has complete control over the target, including access to user accounts, network shares, and the ability to pivot to other systems. The overall risk rating is Critical due to the active foothold and the ease with which the system was compromised.

Top recommendations include immediate isolation of the system, removal of all unnecessary services, patching of vulnerable software, implementation of network segmentation, and deployment of intrusion detection systems.

Methodology

Phase 1

Reconnaissance

Conducted initial reconnaissance on 192.168.50.70 using DNS lookups and WHOIS queries to identify host metadata, network range, and organizational context. Discovered the target is a Docker container on private RFC1918 address space with no DNS records, indicating an internal service or container environment.

Phase 2

Scanning & Enumeration

Performed comprehensive port scanning with nmap to identify 25+ open ports and running services. Enumerated SMB shares, FTP services, web applications, and database instances. Identified critical misconfigurations including anonymous FTP login, SMB null sessions, and exposed admin interfaces like phpMyAdmin and DVWA.

Phase 3

Vulnerability Analysis

Analyzed all discovered services against known vulnerabilities using searchsploit and CVE databases. Identified 19 distinct vulnerabilities including critical backdoors in vsftpd 2.3.4 (CVE-2011-2523) and UnrealIRCd (CVE-2010-2541), high-severity Samba null session issues, and multiple web application vulnerabilities in DVWA and phpMyAdmin.

Phase 4

Exploitation

Exploited the pre-configured port 1524 backdoor using netcat to establish a root shell on the target system. Verified root access via UID 0 and hostname metasploitable. Attempted additional exploitation vectors including vsftpd backdoor and SQL injection, but root access was already achieved.

Phase 5

Post-Exploitation

Enumerated the fully compromised system to identify user accounts, network shares, and lateral movement opportunities. Discovered 30+ user accounts, NFS exports, VNC access, and multiple pivot points including IRC backdoors and web application access. Confirmed complete system compromise with root privileges.

Technical Findings

FINDING-001 Critical

Port 1524 Root Shell Backdoor

Component Port 1524/TCP (bindshell) Phase 2
Description

A pre-configured backdoor on port 1524 provides direct root shell access to the system. This backdoor was actively listening and accessible via netcat connection, granting immediate administrative control without authentication.

Evidence
Verified root shell via 'nc 192.168.50.70 1524' returning root@metasploitable with UID 0/GID 0 privileges.
Impact

Complete system compromise with root privileges. Attacker can execute any command, access all files, modify system configurations, and pivot to other systems.

Remediation

Immediately disable port 1524 and investigate the source of the backdoor. Review all running services and remove unauthorized backdoors. Implement service monitoring and integrity checking.

FINDING-002 Critical

vsftpd 2.3.4 Backdoor (CVE-2011-2523)

Component Port 21/TCP (FTP - vsftpd 2.3.4) Phase 3
Description

The vsftpd 2.3.4 FTP server contains a hidden backdoor that allows remote attackers to execute arbitrary commands. The backdoor listens on port 6000 and is triggered by uploading a malicious user list file.

Evidence
searchsploit identified exploits 'unix/remote/49757.py' and 'unix/remote/17491.rb' for vsftpd 2.3.4 backdoor. Port 6000 was observed as part of the backdoor mechanism.
Impact

Remote code execution with FTP service privileges. Can lead to full system compromise if combined with other vulnerabilities or used to upload additional payloads.

Remediation

Upgrade vsftpd to version 2.3.5 or later which patches this vulnerability. Alternatively, replace with a secure FTP server like vsftpd, ProFTPD with proper configuration, or SFTP/SCP.

FINDING-003 High

Samba Null Session Authentication

Component Ports 139/445/TCP (SMB/CIFS - Samba 3.0.20) Phase 2
Description

Samba configuration allows null (anonymous) sessions, enabling unauthenticated access to shared resources. This permits enumeration of users, shares, and potential data exfiltration.

Evidence
smbclient //192.168.50.70/tmp -N returned 'Anonymous login successful'. SMB signing was disabled and SMBv1 was enabled.
Impact

Attackers can enumerate system users, access shared directories, and potentially exploit SMB vulnerabilities (e.g., EternalBlue) for remote code execution. Enables lateral movement within the network.

Remediation

Disable null sessions in smb.conf with 'map to guest = Never'. Enable SMB signing with 'server signing = required'. Disable SMBv1 and use SMBv2/SMBv3 only.

FINDING-004 High

UnrealIRCd 3.2.8.1 Backdoor (CVE-2010-2541)

Component Ports 6667/6697/TCP (IRC - UnrealIRCd 3.2.8.1) Phase 3
Description

The IRC server contains a backdoor that can be triggered via the /msg command, allowing remote attackers to execute arbitrary commands on the IRC server.

Evidence
searchsploit identified 'linux/remote/16922.rb' exploit for UnrealIRCd 3.2.8.1 backdoor. Admin email admin@Metasploitable.LAN was discovered.
Impact

Remote code execution on the IRC daemon, potentially leading to system compromise. Can be used to establish persistence or pivot to other systems.

Remediation

Upgrade UnrealIRCd to a patched version or replace with an alternative IRC server. Disable IRC services if not required for business operations.

FINDING-005 High

Vulnerable Web Applications (DVWA, phpMyAdmin, Mutillidae)

Component Port 80/TCP (HTTP - Apache 2.2.8) Phase 2
Description

Multiple vulnerable web applications are accessible including DVWA v1.0.7, phpMyAdmin, and Mutillidae v2.1.19. These applications contain SQL injection, command execution, and file inclusion vulnerabilities.

Evidence
Web directories /dvwa/, /phpMyAdmin/, /mutillidae/, /tikiwiki/, /twiki/ all accessible. phpinfo.php exposed system configuration details.
Impact

Attackers can execute SQL queries, retrieve database credentials, execute system commands, and potentially achieve remote code execution through web application vulnerabilities.

Remediation

Remove or secure all vulnerable web applications in production. Implement Web Application Firewall (WAF). Use strong authentication and disable default configurations. Regularly patch web applications.

FINDING-006 High

PHP Information Disclosure (phpinfo.php)

Component Port 80/TCP (HTTP - phpinfo.php) Phase 2
Description

The phpinfo.php file is publicly accessible, exposing full PHP configuration including system paths, environment variables, module configurations, and potentially sensitive data.

Evidence
phpinfo.php accessible at http://192.168.50.70/phpinfo.php revealing system configuration details including allow_url_fopen enabled.
Impact

Information disclosure that aids attackers in crafting targeted exploits. Reveals system architecture, paths, and configurations that can be leveraged for further attacks.

Remediation

Remove phpinfo.php from production systems. Implement access controls for system information pages. Disable unnecessary PHP modules.

FINDING-007 High

MySQL Default Authentication

Component Port 3306/TCP (MySQL 5.0.51a) Phase 2
Description

MySQL database is accessible with default or weak authentication credentials. Multiple databases including dvwa, metasploit, owasp10 are exposed.

Evidence
MySQL service running with databases dvwa, metasploit, owasp10, tikiwiki accessible. phpMyAdmin interface available for database management.
Impact

Attackers can access, modify, or delete database contents. May extract credentials, user data, or use databases as pivot points for lateral movement.

Remediation

Change all default MySQL credentials. Implement strong password policies. Restrict MySQL access to authorized hosts only. Disable remote MySQL access if not required.

FINDING-008 Medium

Apache 2.2.8 Outdated with Multiple Vulnerabilities

Component Port 80/TCP (HTTP - Apache 2.2.8) Phase 3
Description

Apache 2.2.8 is severely outdated and contains multiple known vulnerabilities including mod_proxy and mod_cgi exploits. TRACE method is enabled enabling cross-site tracing attacks.

Evidence
Nikto scan confirmed Apache 2.2.8 version. TRACE method enabled. Multiple CVEs applicable including CVE-2017-9798, CVE-2015-3184.
Impact

Remote code execution through mod_proxy_ajp, mod_cgi vulnerabilities. Cross-site tracing attacks can steal cookies and session tokens.

Remediation

Upgrade Apache to version 2.4.x or later. Disable TRACE method. Disable unnecessary modules. Implement security headers (CSP, HSTS, X-Frame-Options).

FINDING-009 Medium

PHP 5.2.4 Outdated with RCE Vulnerabilities

Component Port 80/TCP (HTTP - PHP 5.2.4) Phase 3
Description

PHP 5.2.4 is severely outdated with multiple remote code execution vulnerabilities. allow_url_fopen is enabled facilitating remote file inclusion attacks.

Evidence
PHP version 5.2.4-2ubuntu5.10 confirmed with allow_url_fopen On. Multiple CVEs applicable including CVE-2012-1823.
Impact

Remote code execution through file inclusion vulnerabilities. Information disclosure and potential system compromise through PHP exploits.

Remediation

Upgrade PHP to version 7.4.x or later. Disable allow_url_fopen. Implement strict input validation and output encoding.

FINDING-010 Medium

Telnet Service Unencrypted Remote Access

Component Port 23/TCP (Telnet - Linux telnetd) Phase 2
Description

Telnet service provides unencrypted remote access, exposing all credentials and data in plaintext. Vulnerable to credential interception and brute force attacks.

Evidence
Linux telnetd service running on port 23. No encryption or authentication hardening detected.
Impact

Credential interception enabling account compromise. Brute force attacks possible. Lateral movement through captured credentials.

Remediation

Disable Telnet service. Use SSH for all remote access. Implement key-based authentication where possible.

FINDING-011 Medium

VNC 3.3 Weak Authentication

Component Port 5900/TCP (VNC - Xtightvnc 3.3) Phase 2
Description

VNC service uses weak authentication with password stored in /root/.vnc/passwd. Vulnerable to brute force attacks and potential denial of service.

Evidence
Xtightvnc 3.3 detected with password file at /root/.vnc/passwd. Resolution 1024x768 with TightVNC protocol 3.3.
Impact

Unauthorized remote desktop access through brute force. Potential system compromise and lateral movement through VNC session hijacking.

Remediation

Disable VNC if not required. Use SSH tunneling for remote access. Implement strong authentication and network-level access controls.

FINDING-012 Medium

SMTP SSLv2 Protocol Vulnerability

Component Port 25/TCP (SMTP - Postfix smtpd) Phase 3
Description

SMTP server supports SSLv2 protocol which is vulnerable to POODLE and BREAST attacks. Allows protocol downgrade and credential interception.

Evidence
SSLv2 protocols detected on SMTP service. Vulnerable to CVE-2014-3566 (POODLE) and CVE-2011-3389 (BREAST).
Impact

Man-in-the-middle attacks enabling email interception. Credential theft and potential email server compromise.

Remediation

Disable SSLv2 and TLSv1.0. Use TLSv1.2 or TLSv1.3 only. Implement certificate validation and secure cipher suites.

FINDING-013 Low

ISC BIND 9.4.2 Outdated DNS Server

Component Port 53/TCP (DNS - ISC BIND 9.4.2) Phase 3
Description

Outdated DNS server vulnerable to zone transfer attacks, cache poisoning, and other DNS-specific exploits. Multiple CVEs applicable to version 9.4.2.

Evidence
BIND 9.4.2 service running. Vulnerable to CVE-2008-1447, CVE-2003-0601 and other DNS vulnerabilities.
Impact

DNS cache poisoning enabling man-in-the-middle attacks. Zone transfer exposing internal network topology. Potential DNS-based attacks.

Remediation

Upgrade BIND to latest version. Disable zone transfers to unauthorized hosts. Implement DNSSEC for authentication.

FINDING-014 Low

RPC/NFS World-Readable Exports

Component Ports 111/2049/TCP (RPC/NFS - rpcbind 2, NFS 2-4) Phase 2

Evidence
rpcbind 2 and NFS 2-4 exports detected. NFS export '/ *' with mountd, nlockmgr services active.
Impact

Unauthorized file access and data exfiltration. Potential system modification through NFS-mounted shares.

Remediation

Restrict NFS exports to authorized hosts only. Implement NFSv4 with proper authentication. Disable NFS if not required.

FINDING-015 Low

Description

HTTP TRACE method is enabled on the web server, enabling Cross-Site Tracing (XST) attacks that can steal cookies and authentication tokens.

Evidence
Nikto scan confirmed TRACE method active on Apache 2.2.8 server.
Impact

Session hijacking through cookie theft. Authentication bypass and potential account compromise.

Remediation

Disable TRACE method in Apache configuration. Implement security headers to prevent XST attacks.

FINDING-016 Low

Directory Indexing Enabled

Component Port 80/TCP (HTTP - Apache 2.2.8) Phase 2
Description

Multiple directories (/icons/, /doc/, /test/, /dav/) have directory indexing enabled, exposing file listings and system structure to attackers.

Evidence
Directory listing accessible on /icons/, /doc/, /test/, /dav/ paths. Web server configured to allow directory browsing.
Impact

Information disclosure revealing system structure, file names, and potential attack vectors. Aids attackers in reconnaissance.

Remediation

FINDING-017 High

SMB Signing Disabled

Component Ports 139/445/TCP (SMB/CIFS - Samba 3.0.20) Phase 2
Description

SMB signing is disabled on the system, enabling man-in-the-middle attacks, session hijacking, and SMB relay attacks.

Evidence
smb-security-message_signing: disabled detected. SMBv1 enabled alongside signing disabled.
Impact

Man-in-the-middle attacks enabling credential theft and session hijacking. SMB relay attacks possible for lateral movement.

Remediation

Enable SMB signing with 'server signing = required'. Disable SMBv1 and use SMBv2/SMBv3 only.

FINDING-018 High

SMBv1 Protocol Enabled

Component Ports 139/445/TCP (SMB/CIFS - Samba 3.0.20) Phase 2
Description

SMBv1 protocol is enabled on the system, exposing it to known vulnerabilities including EternalBlue and other SMB-based exploits.

Evidence
SMBv1 enabled on Samba 3.0.20-Debian server. Legacy protocol with known critical vulnerabilities.
Impact

Remote code execution through SMB vulnerabilities like EternalBlue. Lateral movement and system compromise possible.

Remediation

Disable SMBv1 completely. Use SMBv2 or SMBv3 only. Apply Microsoft security patches for SMB vulnerabilities.

FINDING-019 Medium

Outdated PostgreSQL version with potential weak authentication mechanisms. Expired SSL certificate from 2010-04-16 indicates poor security practices.

Evidence
PostgreSQL 8.3.0-8.3.7 running with SSL certificate expired 2010-04-16. Multiple CVEs applicable to version 8.3.0.
Impact

Database access through weak authentication. Data exfiltration and potential system compromise through database exploits.

Remediation

Upgrade PostgreSQL to latest version. Implement strong authentication mechanisms. Replace expired SSL certificates.
